# Hunter Visual Exploration Prompt

Use this prompt if we want to generate a polished visual reference before implementing another UI iteration.

## Prompt

Design a premium macOS native desktop app called Hunter. It is an AI slacking supervisor with a minimal floating desktop orb and an expanded catch widget.

Visual style:

- Apple native software aesthetic, elegant, minimal, high-end, calm, restrained.
- Light mode, translucent macOS material, soft shadows, white and warm gray surfaces.
- No cyberpunk, no dark AI dashboard, no neon, no heavy gradients, no game-like UI.
- Use SF Pro style typography, generous spacing, subtle rounded corners.
- Main colors: system white, light gray, Apple blue accent, tiny red accent only for caught state.

Primary scene:

- A macOS desktop with a small floating orb near the lower left edge.
- The orb is a 58px translucent white circle with a tiny green status dot and a simple “H” mark.
- When the user is caught on YouTube or Bilibili, the orb expands into a small translucent widget, around 340px wide.
- The widget shows:
  - “Caught on YouTube”
  - a short roast sentence
  - a minimal voice waveform
  - two pill buttons: “Hold Option Space to reply” and “Pause 5 min”

Secondary scene:

- A simple Hunter settings window on the right.
- macOS traffic light controls, narrow sidebar with five items: General, Watchlist, AI, Voice, History.
- General page only shows four rows: work hours, floating widget, reply shortcut, launch at login.
- AI Provider page should look simple: three compact rows for ASR, LLM, TTS, each with provider name, model id, connection status, test button.

Mood:

- Feels like an Apple utility app, not a startup SaaS dashboard.
- Quiet by default, dramatic only at the caught moment.
- Clean enough to ship as a real Mac app.

Canvas:

- 16:10 desktop screenshot composition.
- High fidelity UI mockup.
- No marketing hero text.
- No decorative blobs or mascot characters.
